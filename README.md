# Минэнерго Мониторинг — Android-приложение (v3)

Нативное Android-приложение (Kotlin + Jetpack Compose) для мониторинга
любых гос-сайтов с публикацией документов. Приложение периодически
проверяет заданные пользователем страницы, фильтрует найденные файлы
по правилам и присылает push-уведомления о новых документах.

## Что нового в версии 3

По сравнению с первой версией приложение полностью переработано:

- **Несколько источников.** Можно добавлять, выключать и удалять любые
  сайты через экран "Источники". У каждого свой URL и своя минимальная
  дата публикации.
- **Универсальный парсер.** По умолчанию работает на любом сайте, где
  ссылки на документы соседствуют с датой в формате ДД.ММ.ГГГГ
  (типичные гос-листинги). Для нестандартных сайтов можно задать
  CSS-селекторы и regex даты в "Продвинутых настройках" сайта.
- **Глобальные и локальные фильтры.** Фильтр по подстрокам в названии,
  по расширению файла и по диапазону размера. Глобальный применяется
  ко всем сайтам, локальный — поверх него (документ должен пройти оба).
- **Настраиваемое расписание.** Период проверки от 15 минут до 24 часов
  (минимум — ограничение Android для фоновых задач без постоянного
  уведомления). Окно активных часов теперь отключаемое.

## Главные экраны (нижняя навигация)

1. **Документы** — найденные файлы, кнопки "Проверить" и "Скачать (N)",
   статус-карточка с расписанием и количеством активных источников.
2. **Источники** — список сайтов с переключателями и кнопкой "+".
   Карточка каждого сайта открывает редактор с продвинутыми опциями.
3. **Фильтр** — глобальный фильтр (название/расширение/размер).
4. **Расписание** — переключатель автопроверки, периодичность
   (число + единица: минуты/часы/сутки), окно активных часов.

## Продвинутые настройки парсера для одного сайта

Если сайт имеет нестандартную структуру и универсальная эвристика
не работает, в редакторе сайта можно задать:

- **CSS-селектор ссылок** — например `.docs-list a[href$=".pdf"]`,
  чтобы выбирать ссылки только из определённого блока.
- **CSS-селектор контейнера с датой** — если ссылка и дата лежат
  в общем родителе (например, `.doc-row` или `tr`).
- **Regex для даты** с группой 1, например
  `опубликовано:\s*(\d{2}\.\d{2}\.\d{4})`. Сработает раньше
  стандартных шаблонов парсера.
- **Формат даты** для парсинга, по умолчанию `dd.MM.yyyy`.

## Безопасность

- Скачивание разрешено только с хостов добавленных сайтов
  (динамический allowlist).
- В архивах: защита от Zip-Slip (запрет `..`-путей и абсолютных путей),
  лимит количества файлов (1000), общего объёма распаковки (2 ГБ),
  глубины (10 уровней).
- Чёрный список расширений внутри архивов (`.exe`, `.bat`, `.ps1`,
  `.sh`, `.vbs`, `.js`, `.jar`, `.scr`, `.msi`, `.com`, `.apk`, `.dex`).
- `usesCleartextTraffic="false"`, только HTTPS и HTTP с доменов из allowlist.

## Сборка APK

Через GitHub Actions всё уже настроено: после `git push` в `main`
GitHub автоматически соберёт APK (~5–8 минут) и положит его в раздел
**Actions → последний run → Artifacts**.

Локальная сборка через Android Studio: открыть папку проекта,
дождаться импорта Gradle, **Build → Build APK(s)**.

## Миграция с v1

Старая версия использовала жёстко заданный URL Минэнерго и хранила
данные в ключах `minenergo_state`. При первом запуске v3:

- Создаётся дефолтный сайт "Россети Центр и Приволжье" с тем же URL
  и датой 01.04.2026 (если список сайтов пуст).
- Старые скачанные файлы остаются в `<filesDir>/minenergo_rosseti_cp/`,
  но новые скачивания пойдут в `<filesDir>/downloads/<siteId>/<date>/`.
  Лучше очистить кеш приложения после обновления, если место критично.

## Ограничения

- Минимальный интервал — 15 минут. Для более частых проверок
  потребовался бы foreground service с постоянным уведомлением.
- WorkManager на современных Android может запускаться не точно
  по расписанию (Doze, App Standby). Для надёжной работы
  рекомендуется отключить оптимизацию батареи для приложения.
- История находок не хранится — только последний снапшот.
- Файлы лежат во внутреннем хранилище приложения; при удалении
  приложения они тоже удаляются.
